MainView

On the left is the Map Tree. This is where tilesets are listed. The toplevel branch is the Group, under that is the Category, and under that are the Tilesets.

The Group's label must start with either "ufo" or "tftd" (case insensitive). This determines the palette and the default basepath for all tilesets in that group.

The Category can be labelled whatever.

The Tileset is what gets displayed in the viewer panel. It appears very much like what is seen during gameplay.

Left-click selects a branch, and a right-click opens a context menu that allows the selected branch to be edited. Groups and Categories can be created, re-labeled, or deleted. Tilesets can be created or modified with the Tileset Editor. They can also be deleted.

Above the viewer panel are buttons for
- zoomin
- zoomout
- autozoom
- level up [mousewheel up]
- level down [mousewheel down]
- cut a selected area [Ctrl+X]
- copy a selected area [Ctrl+C]
- paste at a selected tile [Ctrl+V]
- fill a selected area with the currently selected tilepart

The [Delete] key will delete a selected area.

The statusbar below the viewer panel shows
- the current scale factor of the tileset graphic
- the label of the current tileset
- the dimensions (x,y,z or column,row,level) of the current tileset
- the currently selected tile position

The basepath of the currently displayed tileset is shown in the window's titlebar.

In the viewer panel, a left-click selects a tile, and a left-click drag motion will select multiple tiles over a rectangular area.

Autozoom turns off when zoomin or zoomout are used.

The top of the map, north, is to the upper right direction.


File menu

Save All [Ctrl+A] - forces a save of the current Map, Route, and Maptree files.

Save Map [Ctrl+S] - forces a save of the current Map file.

Save Routes [Ctrl+R] - forces a save of the current Routes file.

Save As ... - saves the current Map and Route files as a user-specified path and label. The path cannot be the root directory of a drive. The reason for this is that when you save the Map, the routine finds the parent folder of the chosen folder, then ensures that the Map gets saved in a subfolder called \MAPS and the Routes get saved in a subfolder called \ROUTES. This allows the saved Map to be added to MainView's Maptree while maintaining consistency of design.

Save Maptree [Ctrl+T] - forces a save of the Maptree.

Save Image - saves a .gif of the tileset that's currently displayed in the viewer panel to a specified path.

Resize Map - opens a dialog box for modifying the dimensions of the currently loaded Map.

Map Info - displays a bit of information about the currently loaded tileset.

Regen Occult [Ctrl+O] - to draw all the sprites in the viewer panel faster, any parts that are hidden beneath floors will be ignored, or occulted. But if you strip a floor off of a level above the ground floor this means you could be left looking into a big hole of what appears to be nothing. The tileparts are still there but they've been occulted for faster drawing. Use Regen Occult to make them appear as expected. Also mind that the reverse happens. If a floor is laid down above ground level, and MainView starts lagging, try issuing the Regen Occult command which will stop any sprites that have been hidden from drawing. Occulted tiles are re-calculated every time a Map loads.

Quit [Ctrl+Q] - quits MapView II


Edit menu

Options - opens the Options for MainView. Descriptions for the entries appear at the bottom of the window.

Configurator - opens the Configuration Editor. This is where to specify or modify the paths to your XCOM installation folder(s) for UFO and/or TFTD. It can also replace your Maptree file with a fresh one that's hardcoded. WARNING: doing that would obliterate all customizations that have been done to the Maptree. Alternately, the Configurator can generate a file of the hardcoded, or default tileset-data to a template file. The template would be useful if MapView II had been configured for UFO (or TFTD) and subsequently requires the data for TFTD (or UFO) tilesets. That default data can be copied from the template into your MapTilesets.yml file. Note that the tileset-data for both games will exist in your MapTilesets.yml until the first time that the Maptree is saved, at which point the data for either game that is not configured will be erased. Hint: force a save of the Maptree to get the application to initialize a fraction of a second faster.


Viewers menu

TileView - opens and closes the window that shows the tileparts of the terrains of the currently loaded tileset.

TopView - opens and closes the window that shows an editable, 2-dimensional view of the currently loaded Map. This can be quicker and handier than working directly with the MainView window.

RouteView - opens and closes the window that shows an editable, 2-dimensional view of the currently loaded Routes.

TopRouteView - opens and closes a window that conflates TopView and RouteView as separate tab-pages but in a single viewer.


Help menu

Help - shows some help, some of which may be obsolete. Of particular interest is that on the TileView help-tab, the background colors for tileparts' special properties are tabled.

About - version, build date, you know the drill.
