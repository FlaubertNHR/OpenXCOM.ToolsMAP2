PckView II

PckView is a spriteset editor for XCOM.

PckView II can open, create, or modify standard PCK+TAB files that contain terrain sprites that are 32x40 8-bpp only. It does not work with any other PCK files.

Left-click selects a sprite.

Right-click shows a contextmenu based on the currently selected sprite after opening or creating a spriteset.

  Edit - opens the currently selected sprite in the Sprite Editor. Double-clicking a sprite also brings it up in the editor, as does pressing Enter if a sprite has been selected.
  Add ... - adds a sprite or sprites to the end of the currently loaded spriteset.
  Insert before ... - inserts a sprite or sprites directly before the currently selected sprite.
  Insert after ... - inserts a sprite or sprites directly after the currently selected sprite.
  Replace ... - replaces the currently selected sprite with a different image.
  Move left - swaps a currently selected sprite with the sprite directly before it.
  Move right - swaps a currently selected sprite with the sprite directly after it.
  Delete - removes the currently selected sprite from the spriteset.
  Export sprite ... - exports the currently selected sprite to a file in Windows BMP format.

Adding and inserting sprites requires that their files are 32x40 8-bpp images in Windows BMP format.


File menu

Open Pck file ... - opens a PCK spriteset that contains XCOM terrain sprites.

Create Pck file ... - creates a new PCK spriteset that can contain XCOM terrain sprites.

Save - saves the currently loaded PCK spriteset.

Save As ... - saves the currently loaded PCK spriteset with a different filename.

Export sprites ... - exports all the sprites in the currently loaded spriteset to a directory. The images will be in Windows BMP format.

Quit - exits the application. THERE IS NO PROMPT TO SAVE ANY CHANGES. This is a bare-bones app that requires prior understanding of how XCOM terrain sprites work and why they work as they do, please stay on the ball.


Palette menu

Shows a list of available palettes. Note that sprites do not contain color information - only the 256 palette indices matter. Hence the currently selected palette makes no difference when saving a spriteset. However, the sprites are displayed in PckView with hard-coded color information, which typically make them appear differently than they would while actually playing XCOM.


Transparency menu

On/off - toggles whether or not color index #0 is displayed as transparent. This affects only the display in PckView and does not get saved to the spriteset.


Bytes menu

Show/hide byte table - opens or closes a window that shows the color-indices of all 1280 pixels for the currently selected sprite. The table is arranged in 32 columns with 40 rows, representing the dimensions of the sprite itself, but is not editable.


Help menu

About - version information. Also shows the build-type and currently running OS architecture.
