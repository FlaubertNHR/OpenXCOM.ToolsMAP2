Overview

Every XCOM battlefield-block/ -module is comprised of several files.

    - one .MAP file
    - one .RMP file
    - one or more .PCK files
    - one or more .TAB files
    - one or more .MCD files

To play the game, these files must be located in the subdirectories of XCOM's installation folder.

    \MAPS contains .MAP files
    \ROUTES contains .RMP files
    \TERRAIN contains .PCK, .TAB, and .MCD files

The filenames of the .MAP and .RMP files must be the same. The filenames of the .PCK, .TAB, and .MCD files must also all be the same, but are usually different than the .MAP and .RMP files.

Each map, or tileset, is a 3-dimensional space with width (x/column), height (y/row), and depth (z/level). The .MAP file specifies the dimensions of a map-block aka map-module, and it also contains generic references to tileparts, the specific parts being determined by the terrains that are allocated to the tileset.

Each tileset must also have a routes file. This file may be empty, but usually it contains the x,y,z coordinates of route-nodes, each of which has characteristics that determine how each node may be used during gameplay. Nodes are not visible to a player.

Note that this documentation refers to each Map with all its associated files as a tileset. The three files that make up a terrain are called a terrain, and multiple terrains that have been added to a tileset may be called a terrainset.

MapView directly creates or edits the Map-files, the Route-files, and which terrains are used in any tileset.

It does *not* directly edit Pck-files, Tab-files, or Mcd-files. Pck-files contain sprites and a Pck-file is a spriteset. Tab-files contain indices into its corresponding spriteset for the retrieval of a specific sprite to display as a tilepart. Mcd-files contain records of tilepart characteristics, such as whether a particular tilepart is passable, explosive, blocks fire from spreading, and about 60 other things.

That's almost inarguably what makes XCOM battles so great. The permutations of the ways in which tileparts go together to make tilesets is what MapView is all about.

MapView allows the viewing of spritesets (PCK+TAB files) and terrain records (MCD files), while Volutar's MCDEdit ( https://ufopaedia.org/index.php/MCDEdit ) is one of the generally accepted applications for their modification. There are other methods available such as BB's Toolkit ( http://www.strategycore.co.uk/files/bbs-toolkit/ ) or any decent hex-editor if you're that good.

PckView is included as a decent spriteset editor but, within a spriteset, it can do only pixel-by-pixel editing on the sprites themselves.

McdView is included as a fully functional MCD editor.
