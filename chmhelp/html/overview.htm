Overview

XCOM battlefields are comprised of several files.

- a .MAP file

- a .RMP file

- one or more .PCK files

- one or more .TAB files

- one or more .MCD files

To play the game, these files must be located in subdirectories of XCOM's installation folder.

- MAPS contains .MAP files

- ROUTES contains .RMP files

- TERRAIN contains .PCK, .TAB, and .MCD files

The filenames of the .MAP and .RMP files must be the same. The filenames of the .PCK, .TAB, and .MCD files must also all be the same, but are usually different than the .MAP and .RMP files

Each map, or tileset, is a 3 dimensional cuboid with width (x or col), height (y or row), and level (z or lev). The .MAP file specifies the dimensions of a map-block aka map-module, and it also contains generic references to tileparts, the specific parts being determined by the terrains that are allocated to the tileset.

Each tileset must also have a routes file. This file may be empty, but usually it contains the x,y,z coordinates of route-nodes, each of which has characteristics that determine how the node may be used during gameplay. Nodes are not visible to a player.

MapView directly creates or edits the Map-files, the Route-files, and which terrains are used in any tileset.

It does *not* directly edit Pck-files, Tab-files, or Mcd-files. Pck-files contain sprites and a Pck-file is a spriteset. Tab-files contain indexes into its corresponding spriteset for the retrieval of a specific sprite to display the tilepart. Mcd-files contain data-records of tilepart characteristics, such as whether a particular tilepart is passable, explosive, blocks fire from spreading, and about 60 other things.

That's almost inarguably what makes XCOM battles so great. The permutations of the ways in which tileparts go together to make tilesets is what MapView is all about.

MapView allows the viewing of spritesets (PCK/TAB) and terrain records (MCD), while Volutar's MCD Editor is the generally accepted application for their modification. There are other methods available such as BB's Toolkit and hex-editors.
